## 先从NLP说起——Prompt的起源

图提示学习并不是凭空冒出来的，它是从NLP领域借鉴过来的思路。要理解图prompt，得先理解NLP里发生了什么。

在2020年之前，NLP的主流范式是"预训练+微调"（pre-train & fine-tune）：先在大规模语料上预训练一个大模型（比如BERT），然后针对每个下游任务（情感分类、命名实体识别等）对整个模型做微调。这个方法有效，但有个问题——每个任务都要完整微调一次模型，参数量大、成本高，而且当标注数据很少时效果不好。

2020年GPT-3出来之后，人们发现一个惊人的现象：你不需要微调模型，只需要给模型一段精心设计的"提示"（prompt），模型就能做新任务。比如你想做情感分类，不用微调，只需要把输入改成"这部电影太棒了。这条评论的情感是____"，模型就会填入"正面"。**Prompt的本质是：不改模型，改输入数据的表达方式，让下游任务的格式和预训练任务对齐。**

这就是所谓的"pre-train, prompt, and predict"新范式。它的优势很明显：模型参数冻结不动，只需要学习很少的prompt参数，在少样本（few-shot）甚至零样本（zero-shot）场景下都能工作。

## 图领域为什么需要Prompt？

图神经网络（GNN）面临着和NLP一模一样的困境，甚至更严重。

**困境一：预训练和下游任务之间有鸿沟。** 图上的预训练任务通常是自监督的——比如预测被遮掩的边（edge prediction）、对比学习（GraphCL）等，这些都是图级或边级的任务。但下游任务可能是节点分类、边分类、或者图分类，任务类型完全不同。直接微调效果经常不理想，因为预训练学到的知识和下游任务之间存在gap。

**困境二：标注数据极其稀缺。** 在你关注的欺诈检测场景里，确认的欺诈标签往往不到5%。在新平台上线时几乎没有历史标注。全监督GNN方法（像DGA-GNN、CARE-GNN）一旦标签减少，性能下降很明显。

**困境三：每个任务都要微调整个GNN，成本太高。** 如果有10个不同的图任务，就要维护10个微调后的模型。

图提示学习的核心想法就是：**能不能像NLP那样，冻住预训练好的GNN，只通过修改图的输入数据（加一些可学习的节点、边、特征向量）来适配不同的下游任务？**

## Prompt在图上具体是什么东西？

这是理解整个领域的关键。在NLP里，prompt是一段文本。在图上，prompt是什么？

All-in-One（KDD 2023 Best Paper）给出了一个非常漂亮的答案。它定义了三个层次的图prompt：

**特征层prompt**：给每个节点的特征向量加上一个可学习的向量。相当于在不改变图结构的情况下，调整节点的"描述方式"。数学上就是 x' = x + p，其中p是可学习的prompt向量。

**结构层prompt**：引入一些可学习的"虚拟节点"和"虚拟边"，构成一个小的子图（叫token structure）。这个子图会和原图的节点建立连接，相当于给原图"附加"了一些额外的结构信息。

**任务层prompt**（这是All-in-One最核心的贡献）：不管下游任务是节点分类、边分类还是图分类，都通过一种叫"induced graph"的方法统一转化为图分类任务。对于节点分类，就在目标节点周围构建一个子图来分类；对于边分类，就在两个端点周围构建子图。这样预训练任务和所有下游任务的格式就统一了。

所以整个流程是：

```
原始图 → 加上prompt（特征/结构/任务三层） → 喂给冻住的预训练GNN → 得到预测
```

训练过程中，GNN参数不动，只优化prompt的参数（通常只有几千到几万个参数，远少于GNN本身的参数量）。

## 为什么这个领域会兴起？

三个推动力交汇在一起：

第一，NLP中prompt learning的巨大成功证明了这条路线的可行性，自然有人会想把它搬到图领域。

第二，图基础模型（Graph Foundation Model）的概念在2023-2024年开始兴起。人们希望训练一个通用的图模型，然后通过prompt快速适配到各种不同的图任务和图数据上。All-in-One就是这个方向的开创性工作。

第三，实际应用的需求驱动。在欺诈检测、药物发现、推荐系统等场景中，标注数据稀缺是常态，prompt learning的few-shot能力正好切中痛点。

## 然后ICML 2025那篇论文为什么特别重要？

"Does Graph Prompt Work?"这篇论文做了一件之前没人做过的事——它不是在提出新的prompt方法，而是从理论上分析：**图prompt到底在做什么？**

它的核心结论是：图prompt本质上是一种**可学习的数据变换算子**（data operation）。加特征prompt相当于在特征空间做平移变换，加结构prompt相当于修改图的邻接矩阵。Prompt learning就是在学习一个最优的数据变换，让变换后的图更适合预训练模型去完成下游任务。

这个结论直接为你的DataPrompt项目提供了理论基础——**既然prompt是数据操作，那数据本身的质量就会直接影响prompt学出来的效果。** 如果训练数据里有噪声标签或有毒样本，prompt学到的"数据变换"就会被带偏。这就是为什么需要D-SVARM来估计每个数据点的价值，给高质量数据更大权重。

## 这篇论文要解决什么问题？

一句话概括：**预训练任务和下游任务之间有鸿沟，怎么用"prompt"来弥合？**

传统的"预训练+微调"流程有一个根本矛盾。比如你用 GraphCL 做预训练，它学的是"同一个图的两个增强视图应该在表示空间里靠近"——这是一个图级别的对比学习任务。但下游你可能要做节点分类（节点级）、边分类（边级）、图分类（图级），三种任务的操作粒度完全不同。论文里 Figure 3 画得很清楚：NLP 里不同任务的"任务子空间"重叠度很高（填空、问答、情感分类本质都是预测词），但图领域的节点级、边级、图级操作差异巨大，重叠很少。

## All-in-One 的解决方案：三个核心设计

### 设计一：把所有任务统一成图分类

这是整篇论文最关键的洞见。既然图级任务的任务子空间最大、和预训练目标最兼容，那干脆把节点级和边级任务都"改装"成图级任务。

怎么改装？用 **induced graph**（诱导子图）：

对于**节点分类**：你要分类节点 v，就把 v 的 k-hop 邻居提取出来，形成一个小子图。这个子图的标签就是 v 的标签。这样节点分类就变成了"给这个小子图分类"。

对于**边分类**：你要分类边 (u,v)，就把 u 和 v 各自的邻居都提取出来，合并成一个子图。这个子图的标签就是边的标签。

这样一来，不管原始任务是什么级别的，输入到模型的都是一个个小图，做的都是图分类。预训练用 GraphCL 学图级表示，下游也做图分类——任务格式完全对齐了。

### 设计二：Prompt Graph 的结构

论文把 NLP prompt 和 graph prompt 做了统一抽象，定义了三个组件：

**Prompt Token（提示令牌）**：就是一组可学习的向量 p₁, p₂, ..., p_k，每个向量的维度和节点特征一样。论文实验中默认用 10 个 token。这些 token 会被加到原图节点的特征上：x̂ᵢ = xᵢ + pₖ。

**Token Structure（令牌之间的结构）**：NLP 的 prompt 词之间是线性排列（一句话），但图的 prompt token 之间可以有更复杂的连接。论文提出三种方式：(1) 学习一个可调参数矩阵来决定 token 之间的连边权重；(2) 用 token 向量的点积来决定连边；(3) 完全独立，不连接。

**Inserting Pattern（插入模式）**：怎么把 prompt graph 接到原图上？NLP 里直接拼在句子后面就行，但图没有"前后"的概念。论文的做法是计算每个 prompt token 和每个原图节点之间的点积相似度，超过阈值 δ 就建立连接，并用相似度作为边权重。

所以整个 prompt 实际上是一个**小图**（10 个节点 + 它们之间的边 + 和原图之间的边），被"嫁接"到原始图上。

### 设计三：用 MAML 做元学习初始化

一个 prompt 要同时适配节点分类、边分类、图分类三种任务，初始化很关键。论文用了 MAML（Model-Agnostic Meta-Learning）的思路：

构建多任务 episode，每个 episode 包含三种任务的 support set 和 query set。在 inner loop 中，prompt 参数在 support set 上做几步梯度下降来适配具体任务；在 outer loop 中，用 query set 上的损失来更新 prompt 的初始化参数。目标是找到一个"万能起点"，从这个起点出发，只需要几步微调就能适配任何任务。

整个过程中，**预训练 GNN 的参数始终冻住**，只更新 prompt 参数和 task head 参数。Prompt 大概只有 3K-19K 参数（见 Table 5），而完整 GNN 有 60K-1.5M 参数，所以训练效率非常高。

## 图级任务——为什么要对整个图做预测？

想象一下这个场景：你不是在看单个用户或单条评论，而是发现了一个**刷单团伙**——一群用户在短时间内集中给同一批商品写好评。你把这群用户和他们评论的商品提取出来，形成一个子图。

图级任务就是判断：**这个子图（这个团伙的行为模式）整体上是不是一个欺诈团伙？**

这个判断不能靠看单个节点或单条边，因为团伙里每个个体可能看起来都很正常（这就是 camouflage 伪装），但他们的**整体行为模式**——连接密度、时间分布、评论相似度——作为一个图来看就很异常。

在其他领域也一样：分子图的图级任务是预测这个分子有没有某种药效；论文引用网络的图级任务可能是判断一个研究社区的主题。

## 现在回到你的核心问题：Prompt 对三种任务的优势

你想想这个困境：如果你用传统的 fine-tuning，你预训练好一个 GNN 之后——

要做节点分类，你需要训练一个 **node classification head**（把节点表示映射到类别）；要做边分类，你需要训练一个 **edge classification head**（把两个节点表示拼起来再映射）；要做图分类，你需要训练一个 **graph classification head**（先做 pooling 再映射）。

这三个 head 的输入格式完全不同，参数也互相独立。更麻烦的是，预训练如果用的是 GraphCL（图级对比学习），它学到的表示是为图级任务优化的，直接拿去做节点分类可能效果很差，甚至出现负迁移。

All-in-One 的 prompt 方案怎么解决这个问题？它通过 **induced graph 把三种任务全部变成图分类**。这样：

第一，只需要**一个** graph classification head，三种任务共享；第二，预训练用 GraphCL 学图级表示，下游也是图分类，**任务格式完全对齐**，不存在鸿沟；第三，不同任务之间的差异全部被吸收到了 prompt 里——prompt 负责把不同任务的输入"改装"成适合预训练模型的格式。模型参数冻住不动，只调几千个 prompt 参数。

所以 prompt 的优势不仅仅是"参数少"，更根本的是：**它把"适配不同任务"这件事从"改模型"变成了"改数据"**。模型不动，数据端做变换，这样一个 prompt 框架就能同时服务三种任务。

## 先纠正你对 NLP Prompt 的理解

你跟我聊天输入的那段话，确实是一种 prompt，但那是**人类层面**的 prompt。论文讨论的是**模型层面**的 prompt——在模型"看到"你的输入之前，数据已经被处理成了向量序列。

举个具体例子。假设你要做情感分类，输入是一条评论：

```
"这家餐厅的菜太难吃了"
```

如果是 fine-tuning 的做法：把这句话编码成向量，喂给预训练模型，后面接一个分类器，输出"正面/负面"。

如果是 prompt 的做法：把输入改装成这样：

```
"这家餐厅的菜太难吃了。总的来说，我觉得这家餐厅很[MASK]"
```

让模型去预测 [MASK] 位置的词。如果模型预测出"差"，就对应"负面"；预测出"好"，就对应"正面"。

注意看发生了什么——**你没有改模型，你改的是输入数据**。你往原始文本后面"附加"了一段额外的内容（"总的来说，我觉得这家餐厅很[MASK]"），让下游任务（情感分类）的格式和预训练任务（预测被遮掩的词）对齐了。

现在关键来了。那段附加的文字，在模型内部是什么？是一串**向量**。每个词被 tokenizer 转成一个 token，每个 token 对应一个向量。所以所谓的 prompt，在模型看来就是**一组被插入到原始输入中的额外向量**。

而且后来研究者发现，这些向量不一定要对应真实的词——你可以直接学习一组**可训练的向量**，让梯度下降自动找到最好的 prompt，而不是人工去想"该用什么词"。这就是所谓的 **soft prompt**。

现在你有了正确的理解基础：**prompt = 一组被插入到原始输入中的可学习向量**。

## 现在迁移到图上

有了上面的理解，我来用一个你熟悉的例子——你的虚假评论检测图——来解释这三个概念。

假设你的图里有 5 个用户节点，每个节点有一个 100 维的特征向量（比如用户的注册时长、评论频率等等）。

### Prompt Token：就是"额外的向量"

在 NLP 里，prompt token 是插入到句子里的额外词向量。在图上也一样——**prompt token 就是一组可学习的向量，维度和节点特征一样**。

比如论文默认用 10 个 token，每个是 100 维。你可以把它们想象成 10 个**"虚拟节点"**——它们不是真实的用户，而是模型自己学出来的、用来辅助任务的"幻影节点"。

```
原始图：5 个真实节点，每个 100 维
Prompt：10 个虚拟节点（token），每个也是 100 维
```

这些 token 的值是随机初始化的，然后通过梯度下降来优化。这就是 **p₁, p₂, ..., p₁₀**。

### Token Structure：这些虚拟节点之间怎么连边？

在 NLP 里，prompt 词之间是线性排列的——"总的"→"来说"→"我"→"觉得"……就是一条链。但在图上，这些虚拟节点之间的连接关系可以是任意的。

论文提出了三种方式：

**方式一**：学一个参数 aᵢⱼ 来决定虚拟节点 i 和 j 之间有没有边、边多强。这个参数和 token 本身一起被梯度下降优化。

**方式二**：用两个 token 向量的点积来决定连不连边。如果 σ(pᵢ · pⱼ) 超过阈值 δ，就连边。

**方式三**：最简单——这些虚拟节点之间完全不连边，各自独立。

所以 token structure 回答的问题是：**这 10 个虚拟节点自己之间构成什么样的"小图"？**

```
方式一：虚拟节点之间的边权重是可学习参数
方式二：虚拟节点之间的边由向量相似度决定
方式三：虚拟节点之间没有边（最简单）
```

### Inserting Pattern：虚拟节点怎么接到原始图上？

这是最关键的一步。在 NLP 里，prompt 直接拼在句子的前面或后面。但图没有"前后"的概念，你得决定：每个虚拟节点和原始图里的哪些真实节点建立连接？

论文的做法是：计算每个 prompt token pₖ 和每个真实节点 xᵢ 之间的点积相似度。如果 σ(pₖ · xᵢᵀ) 超过阈值 δ，就在虚拟节点 k 和真实节点 i 之间连一条边，权重就是这个相似度值。

```
对于每个虚拟节点 k 和每个真实节点 i：
  计算相似度 wᵢₖ = σ(pₖ · xᵢ)
  如果 wᵢₖ > δ：连边，权重为 wᵢₖ
  否则：不连边
```

这样做的效果是：**每个虚拟节点会"选择性地"接到原始图中和它最相关的真实节点上**。不是所有虚拟节点都连到所有真实节点——那样太粗暴了——而是通过相似度筛选，实现精准的"嫁接"。

## 三者合在一起是什么？

把这三步组合起来，你得到的就是一个完整的 **Prompt Graph**：

```
原始图（5个真实节点 + 它们之间的真实边）
    ↓ 嫁接
Prompt Graph（10个虚拟节点 + 它们之间的学习到的边 + 它们和真实节点之间的连接）
    ↓ 合并
增强后的图（15个节点 + 所有边）
    ↓ 输入
冻住的预训练 GNN
    ↓ 输出
预测结果
```

整个过程中，GNN 的参数不动。你只优化那 10 个虚拟节点的向量值、它们之间的连接方式、以及它们和原始图的嫁接方式。这些可训练参数加起来也就几千个，远远少于整个 GNN 的参数量。

你现在能理解为什么论文说 prompt 是一种"数据操作"了吗？它没有改模型，它改的是输入给模型的那个图——通过往图上"嫁接"一些精心学习的虚拟节点和边，来引导模型做出正确的预测。

## 你已经理解的部分

节点级预测的流程是：

```
节点 → GNN 聚合邻居信息 → 得到节点向量 h_i（比如 100 维）→ 分类器 → 预测标签
```

每个节点最后都有一个自己的向量 hᵢ，直接拿去分类就行。

## 图级预测多了一步

图级预测的问题是：一个图里有很多节点，每个节点都有自己的向量。但你最终需要的是**整个图的一个向量**，才能对整个图做分类。

所以流程变成了：

```
子图（比如 8 个节点）
    ↓ GNN 消息传递
每个节点得到向量 h₁, h₂, ..., h₈（每个 100 维）
    ↓ Readout（关键的新步骤）
把 8 个向量聚合成 1 个图向量 h_graph（100 维）
    ↓ 分类器
预测这个图的标签
```

## Readout 具体怎么做？

最简单的方式就是把所有节点向量加起来或取平均：

```
h_graph = (h₁ + h₂ + h₃ + ... + h₈) / 8
```

这就是 **mean pooling**。论文里提到的 summation pooling 就是直接求和不除以 8。

用你的虚假评论检测举例：你要判断用户 A 是不是虚假评论者。你提取 A 的 2-hop induced graph，得到一个包含 A 和他 7 个邻居的子图。GNN 在这个子图上做消息传递，8 个节点各自得到一个 100 维的向量。然后 mean pooling 把它们合成一个 100 维的**图向量**，最后分类器看这个图向量，输出"虚假/真实"。

关键在于：**原来你是直接用节点 A 的向量来分类，现在你是用 A 和它邻居构成的整个子图的向量来分类。** 任务本质没变（还是判断 A 是不是骗子），但操作格式变了——从节点分类变成了图分类。

## 先搭建场景

假设在 Cora 数据集上，我们有 6 个具体的二分类任务（实际上会更多，这里简化）：

```

任务3：边分类——"同方向引用" vs "跨方向引用"
任务4：边分类——"高频互引" vs "低频互引"  
任务5：图分类——"理论社区" vs "应用社区"
任务6：图分类——"密集社区" vs "稀疏社区"
```

每个任务都有少量标注数据。对于每个任务，我们把标注数据分成两半：**support set**（用来适配）和 **query set**（用来评估）。你可以把 support set 理解成"练习题"，query set 理解成"考试题"。

然后 prompt 参数 θ 有一个初始值——这个初始值就是我们要通过 meta-learning 找到的"好起点"。

## MAML 的核心：两层循环

### Inner Loop（内循环）：在单个任务上"试着学"

从当前的初始值 θ 出发，针对任务 1，用任务 1 的 support set 做几步梯度下降：

```
θ₁ = θ - α · ∇L_support1(θ)
```

注意，这里得到的 θ₁ 是**临时的**，只是"试一试从 θ 出发学任务 1 能学成什么样"。同样地，从同一个 θ 出发，针对任务 2 也试几步：

```
θ₂ = θ - α · ∇L_support2(θ)
```

针对任务 3、4、5、6 都一样，各自从 θ 出发做几步适配，得到 θ₃、θ₄、θ₅、θ₆。

### Outer Loop（外循环）：评估这个起点到底好不好

现在关键来了。我们用每个任务的 **query set**（考试题）来检验：从 θ 出发学了几步之后，效果怎么样？

```
总损失 = L_query1(θ₁) + L_query2(θ₂) + L_query3(θ₃) + L_query4(θ₄) + L_query5(θ₅) + L_query6(θ₆)
```

这个总损失衡量的是什么？**它衡量的是"θ 作为起点的质量"**。如果从 θ 出发，几步微调后在所有任务的考试题上都表现不错，说明 θ 是一个好起点；如果表现很差，说明 θ 不够好。

然后我们用这个总损失的梯度来**更新 θ 本身**：

```
θ ← θ - β · ∇_θ [L_query1(θ₁) + L_query2(θ₂) + ... + L_query6(θ₆)]
```

注意这里求的是对 **θ** 的梯度，不是对 θ₁、θ₂ 的梯度。因为 θ₁ = θ - α·∇L(θ)，θ₁ 本身是 θ 的函数，所以这里涉及到**梯度的梯度**（二阶梯度），这就是论文公式 (4) 里那个 Hessian 矩阵的来源。

## 用"老师带学生"的比喻串一遍

把整个过程想象成这样：

孙老师（θ）有一套"通用指导策略"。他同时带 6 个学生。

**Inner loop**：孙老师用他当前的策略 θ 分别去指导每个学生几天（几步梯度下降）。学生 1 学成了 θ₁ 的水平，学生 2 学成了 θ₂ 的水平……

**评估**：期末考试。看看每个学生的考试成绩（query set 上的 loss）。

**Outer loop**：如果学生们整体考得不好，孙老师就反思——"**我的初始指导策略**哪里有问题？"然后调整自己的策略 θ。注意，他调整的不是某个学生的状态，而是**自己的通用策略**。

重复这个过程很多轮之后，孙老师就找到了一套最优的初始策略 θ*：从这个策略出发，不管面对什么类型的学生（任务），只要指导几天（几步梯度下降），学生都能考出不错的成绩。

## 先理解问题背景

你有一个预训练好的 GNN，参数已经冻住了。现在你想让这个冻住的模型去做欺诈检测。但是这个模型是用 GraphCL 预训练的，它学到的是"图的通用表示"，并不是专门为欺诈检测优化的。

理想情况下，你可能会想：如果我能把原始图做一些**变换**——比如把一些伪装边删掉、把欺诈者的特征增强一下——变换后的图喂给模型，效果可能会更好。

但问题是：**你不知道该做什么变换。** 你不知道该删哪些边、该怎么改特征。如果你知道的话，那你已经知道谁是欺诈者了，也就不需要模型了。

## Prompt 做的事情

3.5.2 节说的是：**你不需要真的去做那些图变换，你只需要加一个 prompt，就能让模型"以为"你做了那些变换。**

用数学语言说就是 Equation 5：

```
φ*(A, X + p*) ≈ φ*(g(A, X))

左边：原始图加了 prompt 后，冻住的模型给出的表示
右边：原始图做了某种变换 g 后，冻住的模型给出的表示
```

也就是说，加 prompt 产生的效果，和直接对图做变换产生的效果，在模型看来是近似等价的。

## 用一个极端简化的例子来感受

假设你的图有 3 个节点，特征都是 2 维的：

```
用户A（欺诈者）：特征 [0.2, 0.8]
用户B（正常人）：特征 [0.9, 0.1]  
用户C（正常人）：特征 [0.8, 0.3]
```

假设"理想变换 g"是：把欺诈者 A 的特征改成 [0.5, 0.5]，这样模型就更容易区分。但你不知道 A 是欺诈者，所以你没法直接做这个变换。

Prompt 的做法是：学一个 token 向量 p = [0.3, -0.3]，加到所有节点上：

```
用户A：[0.2+0.3, 0.8-0.3] = [0.5, 0.5]  ← 恰好达到了理想效果！
用户B：[0.9+0.3, 0.1-0.3] = [1.2, -0.2]
用户C：[0.8+0.3, 0.3-0.3] = [1.1, 0.0]
```

当然实际情况不会这么完美，所以会有一个**误差项 O**。

## 那 Equation 6 说的是什么？

Equation 5 用的是最简单的 prompt——一个 token 加到所有节点上。这就像你用同一把锤子敲所有的钉子，太粗暴了。

Equation 6 说的是：如果你用 All-in-One 的完整 prompt graph（有多个 token、有 token structure、有 inserting pattern），误差可以**大幅降低**。因为不同的 token 可以选择性地连接到不同的节点上，实现更精细的"数据变换"。

Table 6 的实验数据证实了这一点：单 token prompt 把误差降了 67%，而 10 个 token 的 prompt graph 把误差降了 **95.6%**。

## 为什么这对你的 DataPrompt 项目重要？

这一节的核心结论是：**prompt 的能力取决于它能多好地近似理想的数据变换，而这个能力受限于训练数据的质量。**

如果你的训练数据里有噪声标签——比如一个明明是欺诈者的节点被标成了正常人——prompt 在训练时会试图去"适配"这个错误的标签，学到一个错误方向的数据变换。这就会增大 Equation 6 里的误差项。

D-SVARM 的作用就是：在 prompt 训练之前，先估计每个数据点的价值，把噪声样本的权重降低，让 prompt 从高质量数据中学到更好的数据变换，从而降低误差。

## 先明确：D-SVARM 要嵌入到哪里？

回忆一下 All-in-One 的 meta-learning 流程：

```
每个 episode：
  1. 采样任务
  2. 对每个任务采样数据，划分 support / query
  3. Inner loop：用 support set 的 loss 更新 prompt（几步）
  4. Outer loop：用 query set 的 loss 更新 prompt 初始化
```

数据估值可以切入的位置其实有好几个。我想到了至少四种方案，各有优劣。

## 方案一：在 Inner Loop 的 Loss 上加权

这是最直觉的方案。现在 inner loop 里每个 support 样本对 loss 的贡献是一样的：

```
L = (1/n) · Σ loss(样本i)
```

如果你用 D-SVARM 给每个样本算出一个价值 φᵢ，就可以变成：

```
L = Σ wᵢ · loss(样本i)，其中 wᵢ 正比于 φᵢ
```

高价值样本的 loss 权重大，噪声样本权重小甚至为零。这样 prompt 在 inner loop 中就不会被坏样本带偏。

这个方案简单直接，但有一个问题——**D-SVARM 本身需要大量的 utility evaluation 来估计 Shapley 值，计算成本可能很高。**

## 方案二：用 Prompt 本身作为轻量级 Utility 函数

这是我觉得**最巧妙的方案**，也是你选题报告里提到的思路。

传统的 Data Shapley 要估计 V(S)（在子集 S 上训练模型后的验证集表现），每次都要完整训练一个模型，开销巨大。但在 prompt 框架下，GNN 是冻住的！评估一个子集 S 的效用只需要：

```
V(S) = 在子集 S 上做几步 prompt 更新 → 在验证集上算准确率
```

因为 prompt 只有几千个参数，几步更新非常快。所以 prompt 框架天然就提供了一个**廉价的 utility 函数**给 D-SVARM 使用。这是在传统全模型训练场景下做不到的效率优势。

## 方案三：Episode 采样阶段的数据筛选

不改 loss 函数，而是改**数据采样策略**。在 meta-learning 构建 episode 时，用 D-SVARM 估计每个标注样本的价值，然后：

```
高价值样本 → 更大概率被采样进 support set
低价值样本 → 降低采样概率或直接丢弃
```

这样每个 episode 的 support set 质量都更高，prompt 从更好的数据中学习。

这个方案的好处是**完全不改 All-in-One 的训练流程**，只改数据采样的分布，实现起来最简单。

## 方案四：两阶段流程

第一阶段：先用 D-SVARM 对所有训练数据做一次估值，识别出噪声样本和高价值样本。第二阶段：去掉噪声样本后，用清洗过的数据正常跑 All-in-One 的 meta-learning。

这个方案最简单，但也最"粗暴"——一次性决策，没有在训练过程中动态调整。

## 我的建议：方案二 + 方案一的结合

用方案二的思路来解决方案一的计算成本问题。具体来说：

```
第一步：利用 prompt 的轻量特性，用 D-SVARM 快速估计每个数据点的 Shapley 值
        （每次 utility evaluation 只需要几步 prompt 更新，不需要重训 GNN）
        
第二步：把估计出的 Shapley 值作为权重，嵌入 inner loop 的 loss 函数

第三步：在 meta-learning 过程中周期性地重新估值
        （因为随着 prompt 的更新，数据的相对价值可能会变化）
```

这样你的论文就有一个非常清晰的 story：**ICML 2025 证明了 prompt 是数据操作 → 数据质量影响 prompt 效果 → 用 D-SVARM 估计数据质量 → prompt 的轻量特性让 D-SVARM 的计算成本大幅降低 → 估值结果反过来优化 prompt 训练**。

## 回忆 D-SVARM 的核心流程

3. 一个预算 T：总共允许调用 V(S) 多少次

## V(S) 在图 Prompt 场景下是什么？

给定一个标注节点的子集 S（比如从 700 个里抽了 50 个），V(S) 的计算过程是：

```
第一步：对 S 中每个节点提取 induced graph，构建训练数据
第二步：初始化一个 prompt（可以用随机初始化或某个固定初始化）
第三步：用 S 的数据做几步梯度更新，优化 prompt 参数
第四步：把更新后的 prompt 加到验证集的 induced graph 上
第五步：喂给冻住的 GNN，算验证集上的准确率或 AUC
第六步：返回这个分数，就是 V(S)
```

注意第三步——因为 GNN 是冻住的，prompt 只有几千个参数，所以"做几步梯度更新"非常快。这就是我之前说的，**prompt 框架天然提供了一个廉价的 utility 函数**。

## 然后 D-SVARM 怎么用这个 V(S)？

回忆你的 D-SVARM 算法的主循环：

```
每一轮：
  1. 随机采样一个 stratum ℓ
  2. 从 700 个节点中随机抽 ℓ+1 个节点，构成子集 A
  3. 调用一次 V(A)，得到分数 v
  4. 对 A 中的每个节点 i：更新 φ⁺[i, ℓ]（正向估计）
  5. 对 A 外的每个节点 i：更新 φ⁻[i, ℓ]（负向估计）
```

**每调用一次 V(A)，就同时更新了全部 700 个节点的估计值。**这就是 D-SVARM 相对于传统 TMC-Shapley 的核心优势——最大化样本复用。

## 首先纠正：是验证集，不是测试集

这一点很重要。Data Shapley 的效用函数 V(S) 衡量的是"在子集 S 上训练后，在**验证集**上的表现"。测试集是最终评估用的，整个训练过程中都不能碰。所以：

```
训练数据（700个标注节点）→ 用来训练 prompt
验证集 → 用来计算 V(S)，给 D-SVARM 提供分数
测试集 → 最后写论文报结果时才用，全程不参与
```

## 然后回答你的核心问题：Prompt 怎么更新？

你描述的两种方式其实对应机器学习里两个经典概念。

第一种——把 50 个样本**一起**算 loss，求一次梯度，更新一步，然后再把 50 个样本一起算 loss，再更新一步……这叫 **batch gradient descent**。

第二种——拿第一个样本算 loss，更新一步，再拿第二个样本算 loss，再更新一步，用完就不回头……这叫 **stochastic gradient descent（SGD）**。

在我们这个场景下，用的是**第一种**。因为你的子集 S 可能只有 50 个样本，数据量很小，完全可以一次性全部算 loss。具体流程是这样的：

```
给定子集 A（比如 50 个标注节点）：

第1步：初始化 prompt 参数 θ₀
第2步：对 A 中所有 50 个节点提取 induced graph
第3步：把 50 个 induced graph 都喂给"prompt + 冻住的 GNN"
第4步：得到 50 个预测结果，和真实标签比较，算出总 loss
第5步：对 θ₀ 求梯度，更新 → θ₁
第6步：用 θ₁ 再对同样 50 个样本算 loss，更新 → θ₂
第7步：重复几步 → 得到 θ_K（K 可能就是 3-5 步）
第8步：用 θ_K 在验证集上算准确率，返回 V(A)
```

所以回答你的问题：**同样那 50 个样本会被反复使用 K 次**（每步梯度更新都用全部样本），不是用完就丢掉的。K 不需要很大，3-5 步通常就够了。因为我们不需要 prompt 完全收敛——我们只需要一个粗略的分数来反映"这个子集 A 的数据质量如何"。

## 为什么不需要背景分布？

你原始版本的思路是：抽一个子集 A，再抽一个背景子集 K，然后计算边际贡献 V(K∪A) - V(K)。这需要两次 utility evaluation，而且需要一个背景分布来决定 K 怎么抽。

D-SVARM 的思路完全不同。它根本不算边际贡献。它只算 **V(A) 本身**——一个 coalition 的值。然后通过 φᵢ = φᵢ⁺ - φᵢ⁻ 这个分解，把"差"推迟到最后聚合阶段再算。

用一个简单的例子来感受。假设有 3 个玩家 {1, 2, 3}，你采样到了子集 A = {1, 3}，算出 V({1,3}) = 0.8。

```
传统方法（需要背景分布）：
  还要额外算 V({3}) 或 V({1})，然后做差
  → 两次 evaluation，只更新 1-2 个玩家

D-SVARM：
  V({1,3}) = 0.8，直接用这个值
  → 玩家 1：在 A 里 → 更新 φ⁺[1, ℓ=1]（正向估计）
  → 玩家 3：在 A 里 → 更新 φ⁺[3, ℓ=1]（正向估计）
  → 玩家 2：不在 A 里 → 更新 φ⁻[2, ℓ=2]（负向估计）
  → 一次 evaluation，全部 3 个玩家都更新了
```

## 现在用你的图 Prompt 场景走一遍完整流程

假设有 n = 700 个标注节点，预算 T = 500 次 utility evaluation。

### Phase 1：初始化

```
对每个节点 i（1到700）、每个 stratum ℓ（0到699）：
  φ⁺[i, ℓ] = 0，计数 c⁺[i, ℓ] = 0
  φ⁻[i, ℓ] = 0，计数 c⁻[i, ℓ] = 0
```

这就是两张大表，记录每个节点在每个"coalition 大小"下的正向和负向估计。

### Phase 2：Warmup

这一步确保每个 stratum 至少有一个样本。

```
for ℓ = 0, 1, ..., 699:
  随机抽 ℓ+1 个节点构成子集 A
  用 A 的数据训练 prompt 几步，在验证集上算分 → v
  A 里的每个节点：记录 φ⁺[i, ℓ] = v
  A 外的每个节点：记录 φ⁻[i, ℓ+1] = v
```

但这里有个实际问题——700 个 stratum 的 warmup 就要 700 次 evaluation，已经超过预算 500 了。所以**实际使用中你不需要遍历所有 stratum**，可以只采样一部分，或者把 stratum 做分桶合并。这是工程实现时需要调整的地方。

### Phase 3：Main Loop（核心）

```
while 还有预算:
  随机选一个 stratum ℓ（比如 ℓ = 49）
  从 700 个节点中随机抽 50 个，构成子集 A
  
  ── 调用一次 V(A) ──
  对 A 中 50 个节点提取 induced graph
  初始化 prompt → 用这 50 个样本做 3-5 步梯度更新
  在验证集上算准确率 → v = 0.73
  ─────────────────
  
  A 里的 50 个节点：更新正向估计
    φ⁺[i, 49] = 滚动平均(旧值, 0.73)
    
  A 外的 650 个节点：更新负向估计
    φ⁻[i, 50] = 滚动平均(旧值, 0.73)
  
  预算 -1
```

你看到了吗？**每一次 V(A) 的调用，700 个节点的估计值全部被更新了。** 50 个在 A 里的更新正向估计，650 个不在 A 里的更新负向估计。

### Phase 4：聚合

```
for 每个节点 i:
  φᵢ = (1/700) · Σ_ℓ (φ⁺[i, ℓ] - φ⁻[i, ℓ])
```

最终每个节点得到一个 Shapley 值 φᵢ。值高的是高质量数据，值低的甚至为负的是噪声数据。

## φᵢ⁺ 和 φᵢ⁻ 的直觉理解

你可能会问：为什么最后是 φ⁺ - φ⁻？

**φ⁺[i, ℓ]** 记录的是：在大小为 ℓ+1 的 coalition 中，**包含节点 i** 时的平均表现。

**φ⁻[i, ℓ]** 记录的是：在大小为 ℓ 的 coalition 中，**不包含节点 i** 时的平均表现。

两者的差就是：**有节点 i 比没有节点 i，平均好了多少？** 这正好就是 Shapley 值的含义——节点 i 的边际贡献。只不过这个"差"不是在采样时逐对计算的，而是积累了大量统计量之后在最后一步做减法得到的。

这就是为什么不需要背景分布——**分解 φ = φ⁺ - φ⁻ 取代了显式计算边际贡献的需要。**

## 先区分"问题"和"算法"

**Data Shapley（Ghorbani 2019）** 定义的是一个**问题**：怎么给训练数据点分配价值？它提出了"把每个数据点当作合作博弈中的玩家，用 Shapley 值来衡量贡献"这个 formulation。它用的求解算法是 TMC-Shapley（基于排列采样的蒙特卡洛近似）。

**你的 D-SVARM** 做的事情是：用 Stratified SVARM 的**算法框架**来解决 Data Shapley 的**问题**，并且把它应用到一个全新的场景——**图 Prompt 学习中的训练数据估值**。
